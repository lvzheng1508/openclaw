# Plan Folder

This folder stores execution documents for fork-specific development work.

Rules for this folder:

- Keep approved planning artifacts outside this folder unchanged.
- Add execution-focused documents here before implementation starts.
- Prefer additive, narrow-scope plans when the fork must stay easy to rebase.
- Record any fork-only constraints so later updates from upstream stay manageable.

Current focus:

- Session management for local runtime transcripts in the Control UI.
- Merge-from-upstream and clean-build verification standardization.
